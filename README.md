About Complaint Management System
The Complaint Management System (CMS) is a web-based platform that helps users register, track, and resolve complaints in a structured and efficient manner.

System Overview
The system ensures that every complaint is properly managed, assigned, and resolved with transparency and accountability.

Complaint Assignment
Each complaint is assigned to a specific staff member such as a teacher, warden, mess incharge, or transport staff. Only the assigned person is responsible for resolving that complaint.

Complaint Status Flow
Every complaint follows a fixed status flow:

Created → Complaint is registered
Pending → Waiting for action
In Progress → Work is ongoing
Resolved → Complaint is successfully solved
Once a complaint is marked as Resolved, an email notification is sent to the user. After this stage, the complaint status cannot be changed unless permission is given by the Admin.
Email Notifications
The system automatically sends email notifications:

When a user signs up, a confirmation email is sent saying: "You are successfully registered. Now you can use CMS."
When a complaint is created, an email notification is sent to both the complainer and the resolver.
Whenever the complaint status is updated, a notification email is sent to both the complainer and the resolver for tracking and history.
When the complaint is resolved, a final confirmation email is sent to the user.
OTP Verification System
After a complaint is marked as Resolved, an OTP verification process is used to confirm the resolution.

An OTP is sent to the complainant's email.
The complainant shares the OTP with the resolver.
Only after OTP verification, the complaint is considered fully approved.
Complaint Filters
Users can filter complaints based on different categories:

All
Solved
Unsolved
Delay (complaints not resolved within 7 days)
These filters are available to all users including Admin, Employee, and Student.
Admin Control & Management
The Admin has full control over the system and manages all operations:

Can resolve complaints directly
Can delete any complaint
Can reassign complaints to different resolvers
Manages the complete complaint lifecycle
The system supports multiple users, and there is no fixed limit on the number of Employees, Students, or Admins that can be created.

Complaint Resolution Policy
Every complaint must be resolved within 7 days from its creation date.

If a complaint is not resolved within 7 days, it will be automatically escalated to the Admin through a chain processing system.
Dashboard System
Each user has a personal dashboard where they can view only their own complaints and related information, ensuring privacy and a better user experience.
